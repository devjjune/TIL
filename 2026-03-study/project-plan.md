## 📝 서비스 이름 ( 예시: 일정 관리 서비스 )


- 편의점 통합 서비스
- 기존에 만들어 놓았던 게시판 프로젝트에서 확장

## 🖥️ 핵심 기능


> 데이터의 흐름과 기술적 난이도를 기준으로 나눈 확장 단계
> 

### 1. 외부 데이터 통합 및 정규화 (Aggregator)

→ [데이터 공급] 브랜드마다 제각각인 상품명과 이벤트 규격(1+1, 2+1, 덤 증정 등)을 하나의 표준화된 DB 스키마로 통합

→ 핵심 기술: Spring Batch(주기적 크롤링 및 API 수집), Data Normalization(데이터 정규화)

- 여러 브랜드의 상품과 이벤트를 한눈에 확인
- 브랜드별 이벤트를 피드에 모아서 보여주기
- 이벤트 진행 중인 상품은 아이콘 위에 표시

### 2. 사용자 상호작용 및 커뮤니티 (Engagement)

→ [CRUD 로직] 특정 상품 ID와 연동된 리뷰 시스템을 구축하여 JPA의 연관관계 매핑

→ 핵심 기술: JPA(1:N, M:N 매핑), Querydsl (복잡한 필터링 및 인기순 정렬)

- 상품별 후기 및 댓글 작성
- 유행하는 음식 조합 콘텐츠 제공 (마크정식, 얼박사, 젤리 얼먹 등 / 본인이 직접 만들수도?)
- 좋아요 기반의 카테고리별 인기 순위 (도시락, 음료 등..)

### 3. 실시간 알림 및 개인화 (Intelligence)

→ [성능과 비동기 처리] 수만 명의 사용자에게 실시간 알림 보내기, 유사 상품 계산, 시스템 부하 고려한 설계

→ 핵심 기술: Redis(캐싱 및 알림 대기열), Spring Event 또는 Message Queue(비동기 알림), 추천 알고리즘

- 즐겨찾기 상품 행사 시 알림 전송
- 비슷한 상품 추천(가격, 이벤트, 후기 비교)

### 4. 글로벌 확장성 (Global Standard)

→ [글로벌 표준 준수] 하드코딩된 한국어가 아닌, 시스템 설정에 따라 유동적으로 유연하게 언어를 서빙하는 구조

→ 핵심 기술: Spring MessageSource(i18n), Accept-Language 헤더 처리 로직

- 외국인 관광객을 위한 다국어 지원

### 5. 이후 계획

- 위치 기반 서비스를 통한 조회
    - 편의점 조회: 근처에 어떤 편의점이 있는지 조회
    - 상품 조회: 기업의 API를 통한 재고 연동이 어렵다면 사용자들의 위치 인증을 통해 재고 정보를 업데이트하는 참여형 데이터 시스템 구축

## 🧩 문제 정의


- 각 브랜드별 공식 앱은 잘 되어있지만, 여러 브랜드를 놓고 한눈에 비교할 수 있는 통합 플랫폼이 부족하다.
- 상품의 후기를 빠르게 찾아보거나 공유할 수 있는 커뮤니티가 부족하다.

## 💁‍♂️ 목표 사용자


- 특정 브랜드만 고집하지 않고 손해 없는 구매를 원하는 사용자
- 빠르게 후기나 랭킹을 확인해서 편의점 매대 앞에서의 고민 시간을 줄이고 싶은 사용자
- 유행하는 상품이나 다이어트 등 특정 목적을 가지고 후기나 성분을 체크하고 싶은 사용자
- 한국 편의점에서 만족스러운 구매를 하고 싶은 외국인 관광객 


## 💡유사 서비스 분석


- 다른 분야의 통합 서비스
    - 화해(화장품), 다나와(전자제품), 라스트오더(유통기한 임박 상품) 등

## ✨ 차별점


- 신중하게 고민해서 구매해야 하는 다른 분야의 통합 서비스와 달리 소비자의 빠른 판단을 돕는 것에 초점을 맞춘다.
- 단순 정보 제공을 넘어 커뮤니티 기능을 도입한다.
