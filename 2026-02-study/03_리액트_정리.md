## ⚛️ 리액트(React) 간단한 정리

### 1. 기본 개념: 태그, 엘리먼트, DOM

리액트를 이해하려면 웹의 기본 구성 요소와 리액트의 관계를 알아야 한다. 

- **태그(Tag):** HTML의 문법 단위로 엘리먼트를 정의하는 마크업이다. (예: `<a>`, `<div>`)
- **엘리먼트(Element):** 시작 태그와 종료 태그로 감싸진 실제 객체 덩어리로, 리액트 앱의 가장 작은 단위이다.
- **DOM (Document Object Model):** HTML 문서의 트리 구조다. 리액트는 이 DOM을 직접 만지는 대신 '가상 DOM(Virtual DOM)'을 사용하여 변경된 부분만 효율적으로 업데이트한다.

---

### 2. JSX와 Babel (리액트의 작성 방식)

리액트에서는 자바스크립트 함수 안에서 HTML처럼 보이는 코드를 작성한다. 

- **JSX (JavaScript XML):** 자바스크립트의 확장 문법으로, HTML의 직관성과 JS의 로직을 동시에 사용할 수 있게 해준다.
- **Babel (Preprocessor):** 브라우저는 JSX를 이해하지 못한다. 바벨이 JSX 코드를 `React.createElement()` 같은 순수 자바스크립트 함수로 번역해 준다.

| **구분** | **HTML 방식** | **리액트(JSX) 방식** |
| --- | --- | --- |
| **코드** | `<a href="...">네이버</a>` | `<a href="...">네이버</a>` (함수 내 return) |
| **특징** | 정적임, 가독성 좋음 | 동적임, 로직 포함 가능, 바벨 필요 |

---

### 3. 컴포넌트 (Component): UI의 부품화

리액트는 화면을 '컴포넌트'라는 독립적인 부품으로 쪼개서 관리한다. 

- **대문자 규칙:** 사용자 정의 태그(컴포넌트)는 무조건 **대문자**로 시작해야 리액트가 일반 HTML 태그와 구분할 수 있다.
- **컴포넌트의 가치:** 한 번 만들어두면 여러 곳에서 재사용이 가능하며, UI와 비즈니스 로직을 하나로 묶어 관리하기 편하다.
- **최상위 엘리먼트 규칙:** 리액트는 반드시 **하나의 부모 태그**로 감싸진 구조만 렌더링할 수 있다. (주로 빈 태그 `<></>`인 Fragment 사용)

---

### 4. 리액트의 핵심: Props와 State

리액트 앱이 살아 움직이게 만드는 두 가지 핵심 데이터 관리 방식이다. 

- **Props (속성):** 부모가 자식에게 전달하는 **'읽기 전용'** 데이터이다. (외부에서 들어오는 값)
- **State (상태):** 컴포넌트 '내부'에서 관리되는 유동적인 데이터이다.
    - **리렌더링:** State 값이 변하면 리액트가 이를 감지해 화면을 자동으로 다시 그려준다. 이것이 리액트가 '반응형(Reactive)'인 이유이다.

---

### 5. 요약: 리액트는 왜 쓰는가?

1. **선언적 UI:** "어떻게(How)" 바꿀지 일일이 명령하지 않고, "무엇(What)"을 보여줄지 상태(State)만 정의하면 리액트가 알아서 그려준다. 
2. **가상 DOM:** 바뀐 부분만 계산해서 실제 DOM에 반영하므로 성능이 매우 뛰어나다. 
3. **컴포넌트 기반:** 코드의 재사용성과 유지보수성이 비약적으로 상승한다.
