### 🧠 배운 점 
- [자바 프로젝트 생성 과정]
  - GitHub에서 저장소를 만든 뒤 IDE에서 열면 자동으로 clone되어 로컬 작업이 가능하다.
  - GitHub 주소를 clone하면 IDE를 바꿔도 동일한 프로젝트를 사용할 수 있다. (VSCode &rightarrow; IntelliJ)
  - Gradle 등 빌드 도구는 기본적으로 `src/main/java` 구조를 사용하며, 이 구조를 따르면 IDE에서 자바 프로젝트로 자동 인식된다.
  - `Sources Root`에 등록되어 있어야 IntelliJ가 자바 소스 코드 폴더로 인식한다.

- [데이터 구조 **JSON** vs **CSV**]
  - **JSON:** 계층적 구조로, 중첩된 객체·리스트 표현이 가능하며 복잡한 데이터(예: 노래 + 아티스트 + 태그)를 다루기에 적합하다.
  - **CSV:** 단순한 행·열 형태의 2차원 표 구조로, 숫자나 문자열 기반의 목록 데이터를 저장하거나 분석할 때 효율적이다.

### 💬 회고 
- **자바 프로젝트 설정에 하루 대부분을 사용했다.**
  - GitHub에서 먼저 저장소를 만든 뒤 clone했더니,
Sources Root, Gradle, 패키지 폴더 등을 전부 수동으로 설정해야 했다.
  - 다음부터는 **IntelliJ에서 프로젝트를 먼저 생성하고 → 이후 GitHub에 push**하는 방식이 훨씬 효율적일 것 같다.

- **`Sources Root` 설정 실수**
  - 패키지명이 계속 `package findmyvibe;`가 아닌 `package main.java.findmyvibe;`처럼 전체 경로가 다 떠서 그 원인을 한참 찾았다.
  - 원인은 java 패키지만 Sources Root로 지정했어야 했는데 src 전체를 지정해버린 것이었다. 
  - 이렇게 하면 src 밑의 모든 폴더(main/java/...)가 전부 패키지명에 포함된다고 인식된다.
  - Sources Root는 **자바 코드의 시작점**이자 **package 선언의 기준점**이므로, **실제 패키지명이 시작되는 폴더(java)를 Source Root로 지정해야** 한다.
